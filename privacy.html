Simple Summary: We built Cyanix AI with privacy as the foundation. The bot doesn't store any data, doesn't track you, and doesn't share information with anyone (except the AI providers you choose to use).

3. Information Processing

3.1 Message Processing

When you interact with the Bot:

  • Real-time Processing: Messages are processed in real-time and immediately discarded
  • Selective Processing: Only messages where the Bot is directly mentioned or commands are used are processed
  • No Storage: Messages are not stored, logged, or saved in any database
  • Temporary Memory: Conversation context is temporarily cached in memory during active chats but is cleared when the conversation ends
  • Channel Access: The Bot only has access to messages in channels where it has been invited

Technical Detail: The bot uses Discord.js with Message Content Intent enabled. This is required for the bot to read messages when mentioned. The bot cannot read messages in channels where it doesn't have access.

3.2 Third-Party AI Services

The Bot uses the following AI services. Please review their privacy policies:

  • Groq API: https://groq.com/privacy
  • Anthropic Claude API: https://www.anthropic.com/privacy

Important: When you use the Bot, your messages are sent to these AI services to generate responses. These services have their own privacy policies and data handling practices. We recommend reviewing their policies if you have concerns.

Data Flow: Your message ‚Üí Discord ‚Üí Your bot instance ‚Üí AI Provider (Groq/Anthropic) ‚Üí Response ‚Üí Discord ‚Üí You. No data is stored at any point in our infrastructure.

4. Self-Hosting and Data Control

The Bot is designed to be self-hosted, giving you complete control:

  • Your Infrastructure: All data stays within your control when you self-host
  • Your API Keys: You configure your own API keys with AI providers
  • No External Servers: No data is sent to our servers (we don't have any)
  • Code Transparency: You can audit the complete source code
  • Modification Rights: You can modify the code to meet your specific needs

Self-hosting is the recommended deployment method for maximum privacy and control.

5. Data Security

While we don't store data, we implement security measures:

  1. No Database: No persistent storage means no database to secure
  2. Local Storage: API keys are stored in your local environment variables
  3. Open Source: Full code transparency allows security audits
  4. Regular Updates: We release regular security updates
  5. Dependency Security: We monitor and update dependencies for security

Security Best Practices: When self-hosting, ensure you follow standard server security practices, including keeping your server updated, using firewalls, and securing your Discord bot token.

6. Children's Privacy

The Bot is not intended for users under 13 years of age. We do not knowingly collect personal information from children under 13.

If you are a parent or guardian and believe your child has provided personal information, please contact us through GitHub issues to have it removed.

Discord's Age Requirement: Discord requires users to be at least 13 years old. The Bot follows Discord's policies regarding user age.
